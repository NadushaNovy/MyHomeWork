First level: 
1. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

  git checkout <file>


2.Как вернуть файл из индекса в рабочий каталог?

  git restore --staged <file>

3.Как отменить ошибочный commit ?

  git reset HEAD^
  git reset HEAD~1

4.Как создать новую ветку и переключится в нее?
      git checkout -b <name>

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
    fast-forward merge это простое слияние, которое по сути передвигает указатель ветки вперед.
Такое слияние возможно, если новая ветка начинается от последнего коммита главной ветки.

   non fast-forfard merge это слияние при котором автоматически создаеться дополнительный коммит объединяющий две ветки.
Это происходит, когда ответвления идет не от последнего коммита главной ветки.
 
6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
      При изменении одного и того же файла в разных ветках при слиянии этих веток может возникнуть конфликт, так как Гит не понимает какие из разных изменений нужно оставить.
Для решения конфликта мы должны вручную в файле(в коде) отредактировать изменнения, оставить нужные,сохранить файл и закомитить.


7.Можно ли удалить ветку? Как? 
  Можно: git branch -d <name>
